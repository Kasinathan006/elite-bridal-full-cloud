2.3 Authentication System

Auth Flow — Step by Step
Client sends phone number to POST /auth/send-otp
Backend calls MSG91 API — sends 6-digit OTP to number
OTP stored in Redis with key otp:{phone} TTL 300s (5 min)
Client sends phone + OTP to POST /auth/verify-otp
Backend checks Redis — if match: create/find user in DB
Backend signs access token (JWT, 15min TTL) + refresh token (JWT, 30 days)
Access token returned in response body, refresh token in HTTP-only cookie
Client stores access token in MMKV, sends in every request header
On 401: client auto-calls POST /auth/refresh — uses cookie — gets new access token

5.1 MVP Features — Integration Steps

Feature 1-4: Profile Core
npx prisma migrate dev -- creates profiles table with all fields
POST /profiles — validates with Zod, creates profile, generates slug with slugify npm
GET /[username] in Next.js — getServerSideProps fetches profile from API, returns SEO meta tags
Meilisearch sync: on profile create/update, BullMQ queues profile-index-sync job
Profile completeness score: computed field — count non-null required fields, divide by total

Feature 7: Skill Tags
Seed 60+ skills into DB: npx prisma db seed -- runs prisma/seed.ts
GET /skills — returns all skills grouped by category for UI chip selector
POST /profiles/me/skills body: { skillIds: ['clmk...', ...] } — upsert profile_skills
Meilisearch re-index after skill update — skills[] is filterable

Feature 16: Open to Bookings
Add isOpenToBookings, openToBookingsNote to profiles table in Prisma schema
PATCH /profiles/me body: { isOpenToBookings: true, openToBookingsNote: 'Booking June-Oct 2026' }
Meilisearch: isOpenToBookings is filterable — search filter ?filter=isOpenToBookings=true
UI: profile photo has SVG circle ring (react-native-svg) — green = available, gray = unavailable

Feature 20: Follow System
Create follows table — composite PK (followerId, followingId)
POST /profiles/:id/follow — insert follow row, increment profile.followerCount (Prisma transaction)
BullMQ job: push notification via Firebase FCM to followed artist
BullMQ job: fan-out — add followed artist's recent 10 posts to follower's post_feed_items
GET /feed — SELECT from post_feed_items WHERE userId = currentUser ORDER BY score DESC LIMIT 10

Feature 23: Home Feed
post_feed_items table: (userId, postId, source, score, createdAt)
On new post: BullMQ post-fanout job — INSERT INTO post_feed_items for each follower
Feed score = (recency_weight * 0.5) + (author_engagement_rate * 0.3) + (geo_match * 0.2)
Cold start (new user, no follows): return trending posts from GET /discover/trending
Cursor-based pagination: ?cursor=lastPostId — no OFFSET (OFFSET is slow on large tables)

Feature 25: Portfolio Post
Client calls GET /upload/sign — Fastify returns signed Cloudinary upload params
Client uploads image directly to Cloudinary using signed params — no image bytes hit backend
Client gets back Cloudinary secure_url — sends to POST /posts body: { type:'IMAGE', mediaUrls:[url], caption }
Backend creates post record, queues fan-out job, queues Meilisearch index job
React Native: react-native-image-viewing for full-screen gallery swipe

Feature 34: Hashtags
On post create: parse caption with regex /#(\w+)/g — extract tags
Upsert into hashtags table — increment postCount — insert into post_hashtags junction
BullMQ job: trending-hashtags-refresh — every 6 hours, compute top 20 hashtags into Redis
GET /hashtags/trending — return from Redis cache

Feature 41: Artist Search
Meilisearch index setup: npx ts-node scripts/setup-meilisearch.ts — creates profiles index with settings
GET /search/profiles?q=airbrush&city=Chennai&priceMax=20000 — Fastify proxies to Meilisearch
availableDate filter: first query Postgres availability_slots, pass matching profileIds to Meilisearch filter
Response: Meilisearch returns profiles with highlighted matching text for UI

Feature 48: Booking Inquiry
POST /bookings body: { artistProfileId, desiredDate, eventType, eventCity, message, serviceId? }
Create booking record status=INQUIRY — Prisma transaction
BullMQ: push FCM notification to artist + WhatsApp message via Interakt API
Artist inbox shows booking card: date, event type, client name, message — Accept / Decline / Counter
PATCH /bookings/:id/confirm — status -> ACCEPTED — notify client via push + WhatsApp

Feature 58-59: Messaging & Notifications
Socket.io server: on connection, join room user:{userId} using JWT from auth header
POST /messages/threads/:id/messages — save to DB, emit to Socket.io room thread:{threadId}
Notifications table: INSERT on every event type — emit to Socket.io room user:{userId}
FCM: firebase-admin.messaging().send({ token: device_token, notification: {...}, data: {...} })
Badge count: return unread notification count in every GET /notifications response header
